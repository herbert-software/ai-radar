## 上下文

5b 在 5a（`model-radar-catalog`：11 张 `mr_*` 表 + Zod 闸 `src/db/mr-schema.zod.ts` + 迁移 0008）之上建**录入与保鲜流水线**。5a 把写契约（`mr_review_flag` 单行翻转、`mr_price_history` append-only、Zod 合法性闸、`mr_plan_sources` 定位边、junction 各带 `last_checked`）定义为 spec 需求并测试，但**无生产写路径实现它们**。5b 落地写路径 + 三档抓取变更检测 + 事件流对接 + 陈旧度排程 + 人工 dispose 面。

约束（`docs/model-radar-tech-plan.md` 已锁决策）：**保鲜回路先于 UI**；**抓取只做变更检测、只打「待复核」、绝不自动改 `mr_*` 权威事实**；事实靠人工策展，状态由程序和 DB 保障，不交 LLM。技术栈以 `config.yaml` 为准。复用范式：`src/pipeline/alert-queue.ts`/`alert-scan.ts`（BullMQ「独立队列 = `QUEUE/JOB` 常量 + `create*Worker` + `defaultJobOptions{attempts,backoff,removeOnComplete/Fail}` + **不嵌入 runDailyWorkflow** 并列独立」）、`src/collectors/`（per-source extractor + DI fetch + 注入桩）、`src/kb/store.ts`（ON CONFLICT 写）。

5a 既有事实（落地必须对齐）：`mr-schema.zod.ts` 的 `mrPlanWriteSchema` 是 **partial**（仅 category/currentPrice/currency/sourceConfidence + 同生同灭 refine）；8 个独立 enum schema 存在但**未组合成各表的写校验器**；词表注明「桶2 样例集，limit_type 缺 credit/fast_pass、currency 待扩，留 5b 随录入扩」。`mr_review_flag.target_type ∈ {plan,source,vendor}`（**无 junction**）。`mr_price_history` 列 = `plan_id/old_value/new_value/currency/changed_at/source_url/source_confidence`（无 source_id/raw_payload）。`ai_news_events` 无 event_date（用 `published_at`），有 `merged_into` tombstone（所有下游读必排）。

## 目标 / 非目标

**目标：** 把 5a 写契约接进生产（Zod 校验 / 单行翻转 flag / append-only / 定位边）；已核 8 家全桶可录入；ai-radar 事件触发复核；三档抓取变更检测**只打标不改事实**；陈旧度排程覆盖**所有事实表**；人工 dispose 闭环；抓取子系统**安全**（SSRF/沙箱/裸请求/快照）。

**非目标：** 5c 快照/API、5d 比价页、5e 推荐器；万能爬虫/无头集群；登录绕过；改 5a `mr_*` schema 语义；漂亮录入后台；渠道/代理转售包；乐观并发版本列（单写者策展不需要，留多写者时再加）。

## 决策

### 写契约

**D1：录入与抓取后人工确认两条写路径统一过 Zod 闸；5b 自建 6 个 per-table 组合写校验器。** `src/mr/ingest/` 的 `upsert*` 在写任一 `mr_*` 前，对其涉及的有限值字段调对应 `mr-schema.zod.ts` schema，`mr_plans` 调 `mrPlanWriteSchema.refine`。5a 只给 8 enum + partial plan schema，**5b 须为 `upsertPlan/Model/PlanLimit/PlanClient/PlanModel/PlanSource/Source` 七个 per-table writer 各建组合写校验器，外加改价路径**（`recordPriceChange` 写 `mr_price_history` 前复用 `mrSourceConfidenceSchema`+`mrCurrencySchema` 校验 provenance）= 共 8 个校验落点（复用 8 enum，新增 limit/client/model/source 行的组合 schema + `mr_models.family` 小写 transform）。`mr_plan_models` 是带 `source_confidence` 的断言事实表，**不可漏**（否则模型兼容事实绕过 Zod/冲突打标）。**全桶录入需扩 `mr-schema.zod.ts` 词表**（limit_type 增 credit/fast_pass、currency 增所需 ISO 4217）——这是 5a 文件自留的 5b 扩点（**扩值不改语义，不算越界改 5a**；新增 `mr_*` 列才走单独迁移记为 5a 修改）。

**D2：ingest 区分 identity-upsert 与 fact guarded-write，禁止盲覆盖事实。** `upsertVendor/Model`（identity，唯一键冲突幂等）vs `upsertPlan/PlanLimit/PlanClient/PlanModel`（事实表，含兼容 junction）。**机制**（无乐观版本列，单写者策展）：fact 写 = `INSERT … ON CONFLICT DO NOTHING RETURNING` → RETURNING 空则读既有行**数值归一逐字段比对事实字段** → 相同=幂等 no-op、不同=返回 `{conflict,field,existing,incoming}` + 打 flag，**永不 `onConflictDoUpdate` 盲覆盖事实字段**；二次读容 0 行（单写者前提下视竞态 no-op，绝不 NPE）。**各表事实字段**（lint/test 据此机械判「盲覆盖」）：plan=`current_price/currency/source_url/source_confidence` + **`category`**（必带分桶 facet；5a 唯一键 `(vendor_id,name)` 不含 category，故同 `(vendor_id,name)` 重录但 category 异**必须打 conflict flag、不可静默 no-op**）、limit=`value/source_url/source_confidence`（**`window` 是 5a 唯一键 `(plan_id,limit_type,window)` 组件、非比对事实**——同 plan 同 limit_type 异 window 是不同限额行非冲突；「一 limit_type 一 window」若需强制属 5a 改键，out-of-5b-scope）、client/model junction=`source_confidence/source_url`；`last_checked` 是可自由刷新的 provenance（非事实字段，刷新不算盲覆盖）。**价格路径例外（授权事实更新，非盲覆盖）**：existing-plan 的价格及对应 provenance 变更**只经 `recordPriceChange`（D4）同事务更新**——它是唯一授权 `.set()` plan `current_price/currency/source_url/source_confidence/last_checked` 的入口；D2 的「禁盲覆盖」专指 `upsertPlan` 在 `ON CONFLICT` 冲突分支对这些事实字段**禁 `.set()`、只返回+打 flag**。lint/test 规则即：禁 `upsert*` 冲突分支 `.set()` 事实字段，`recordPriceChange` 例外。

**D3：`mr_models.family` 写前小写归一。** 5a 显式移交 5b 的契约（catalog spec L50 / zod L69）：`upsertModel` 写前 `family.toLowerCase()` 归一，防 `GLM`/`glm` 因 `UNIQUE(vendor_id,family,version)` 大小写敏感误分裂、破坏 5a 去重。

**D4：单一改价入口 + current=latest 从 history 推导 + 同事务刷 provenance/freshness。** 两个 API：**公开** `recordPriceChange(planId,newValue,currency,provenance)` 无外层事务时自身 `db.transaction(tx => _recordPriceChangeTx(tx,…))`；**内部** `_recordPriceChangeTx(tx: TxLike, …)` **只接已开事务句柄 `TxLike`（非 `DbLike|TxLike`）**——本入口多语句且 `FOR UPDATE` 须跨语句持锁，若注入顶层 `DbLike` 锁会在语句结束即释放留 TOCTOU；故 5c 复用时必须传**已开**外层 tx（区别于 `claimRecord` 单语句可接 `DbLike|TxLike`）。同一 tx 内：① `SELECT current_price/currency/provenance … WHERE id=planId FOR UPDATE`（锁行取 old_value；plan 不存在则报错——建行是 `upsertPlan` 职责）；② 写前过 Zod（`source_confidence`/`currency` 枚举闸）；③ **无价变捷径**：若 **`current_price` 非 NULL** 且 `Number(newValue)===Number(current_price)` 且 currency 同 → 仅刷 `mr_plans.source_url/source_confidence/last_checked`（provenance 再核，**不 append no-op 价行**），return（`current_price IS NULL` = needs_login_recheck 占位无价，首个真价走真追加，不被 `Number(null)→0` 误判跳过）；④ 否则真价变：`changed_at = clock_timestamp()`（**拿到行锁后由 DB 生成，非 `now()`/`transaction_timestamp`**——否则 D16 注入的长外层 tx 可在他人已提交改价后插更早 `changed_at` 致 current 与 MAX(changed_at) 倒挂、且同外层 tx 二次调用共享 `now()` 误入同刻冲突），`INSERT mr_price_history(new_value, old_value=改前 current, currency, changed_at, provenance) ON CONFLICT(plan_id,changed_at) DO NOTHING RETURNING id`；⑤ **RETURNING 非空**（真追加）→ `UPDATE mr_plans SET current_price/currency=新值, source_url/source_confidence=provenance, last_checked=now()`；⑥ **RETURNING 空**（同刻冲突，clock_timestamp 下仅并发同微秒罕见）→ 读既有行**数值归一比对 `(new_value,currency)` 元组**（`Number()` 比 new_value、currency 直比——`mrCurrencySchema` 已全大写无需折叠；**同数额异币种=元组异**）：元组异=`price_history_conflict`+打 flag（同事务复用已持 plan 锁）不动 current；元组同=幂等仅刷 last_checked；二次读容 0 行（仅自开 tx 单写者竞态分支可能 0 行；injected-tx 已持锁自写可见、0 行不可达——纯防御）。`upsertPlan` 改价**必须委托**此入口，禁裸改 `current_price`。`mr_price_history` append-only。

### 保鲜回路（闭环）

**D5：`mr_review_flag` 单行翻转写契约（无 setWhere）+ 检测器层 stale-retry 防护。** 打/翻标 = 单语句 `INSERT … ON CONFLICT(target_type,target_id) DO UPDATE SET status='pending', reason=excluded.reason, opened_at=now(), resolved_at=NULL`——**无 setWhere**（pending 时也刷 reason，区别于 `kb/store.ts` 守 terminal `success`；resolved 后重开是预期，`opened_at` 重置为 now 是单行可变标的有意行为，对齐 5a）。**stale-retry 不靠 CAS guard 而靠检测器幂等**（见 D7）：检测器只在 fingerprint **真变**时打标，stale 重试比对到「已更新的 fingerprint」→ 无变化 → 不打标，故已 resolve 的 flag 不被旧 job 无条件重开。

**D6：人工 dispose 最小面（闭环回写）。** `resolveFlag(target)` = plain `UPDATE status='resolved', resolved_at=now()`（5a spec L110）。最小 dispose 面（脚本/函数，非后台）：`listPendingFlags(by target_type/age，**age = `opened_at`（最新触发/重开时刻，配 D5 无条件 `opened_at=now()`）**）+ `markChecked(target)`：resolveFlag + **按标的粒度刷 last_checked 且同一事务**（防 resolve 成功但 last_checked 未刷致下轮 D9 重开）——source 标的刷 `mr_source.last_checked`；**plan 标的刷 `mr_plans.last_checked` 及其全部 child 事实行**（`mr_plan_limits`/`mr_plan_clients`/`mr_plan_models`）的 `last_checked`。`ponytail:` **全刷 child 是刻意权衡**——为闭合「junction 触发 plan flag → 只刷 mr_plans → D9 重扫 child → 复打标跑步机」（R2），代价是窄因复核（如只看了价格）会把没看的兄弟 child 也盖成 fresh、掩盖其真陈旧；**掩盖有界**（≤1 个陈旧窗口 = D9 配置阈值，默认 ~30 天，后 D9 必重触发，非永久），单写者策展下「复核 plan = 复核整行集」可接受；要 per-row 精度须给 `mr_review_flag` 加 child 引用列（改 5a schema）= 越界留后。否则 flag 只进不出、陈旧度反复重打标。

**D7：三档抓取 = per-source extractor，检测器原子 compare-and-update fingerprint，只打标不改事实。** `src/mr/scrape/`：
- 抽「价格/额度区域」归一文本（每源一小段正则/切片，不引 cheerio）→ `content_fingerprint = sha256`。
- **原子**：同事务比对 `mr_source.content_fingerprint`，**真变**时才 `UPDATE content_fingerprint + last_checked` + 经 `mr_plan_sources` 定位覆盖 plan → 逐个打 flag（**定位空集合则给 source 自身打 `target_type='source'` flag**，保证页面变动永不被吞）。无变化只刷 `last_checked`，不打标。**绝不改 `mr_*` 价格/限额/兼容事实**。
- **结构保证「只 propose」**：事实 writer 全部归 `src/mr/ingest/`——`upsert*` **及 `recordPriceChange`**（它改 `mr_plans.current_price` 是事实写，必须与 upsert 同目录，**不可留在抓取链可达的 `src/mr/write/`**）；flag CAS / fingerprint / last_checked 更新器归 `src/mr/write/`，是抓取链**允许** import 的唯一写入口。`src/mr/scrape/` 与 `event-consumer` **禁止 import `src/mr/ingest/`**（eslint `no-restricted-imports`，含 `recordPriceChange`），把「抓取改不了事实」从人的纪律降为 lint 错误。

**D8：事件消费者 = 独立 cron 队列，键 `published_at` 闭区间 + 排 tombstone。** `src/mr/freshness/event-consumer.ts`：**独立 BullMQ 队列**（cron 时点在每日 workflow 产出事件之后，**不嵌入 `run-daily-workflow.ts`**，对齐 alert-queue 范式）；候选门 = **闭区间 `startOfDayInTimeZone(now, windowDays-1) <= published_at <= now`**（windowDays env 可配但**必须 `>=1`**——env 校验拒 `0`：`windowDays=0` 会令 `daysBack=-1`、下界算成明天 0 点、区间空集**静默停打标**，且与 alert-scan「0=不限窗口」心智相反；对齐 alert-scan L216-220 的闭区间而非「当天」单日：**下界**防回填批量误打标，**上界 `<= now` 绝不可省**——拦 AI 推断的未来 `published_at` 免费越过下界刷屏；`published_at` nullable，`gte/lte` 对 NULL 返 false 故无日期事件自然排除，明确接受），**排除 `merged_into IS NOT NULL` tombstone**；匹配 = `mr_vendors.normalized_name`（已小写归一）vs 事件 `representative_title/summary_zh/headline_zh`（**三列均 nullable，任一为 NULL 跳过该列，不对 NULL 归一/拼接**）两侧归一后子串/词边界 + 价格/模型关键词常量（`const REVIEW_TRIGGER_KEYWORDS`，校准旋钮）。命中 → 给该厂商 plan 经 D5 单语句 CAS 打标（**不做「写前查 status」预检**——CAS 本就幂等收敛单行，预检是 read-then-write TOCTOU，会与人工 resolve 竞态**丢真实事件**；already-pending 重命中刷 reason/opened_at 良性、非正确性损失）。**只写 flag、不改事实**。多 plan 打标 **per-target 独立**（每 CAS 自治 autocommit，失败隔离，不裹批事务）。

**D9：陈旧度排程覆盖所有事实表 + NULL 语义。** `last_checked` 超阈值（默认 30 天 env 可配）的 **`mr_source` 与各事实表（`mr_plans`/`mr_plan_limits`/`mr_plan_clients`/`mr_plan_models`）** 进复核：source 超期 → source flag；junction/limit 超期 → **给其所属 plan 打 plan 级 flag**（reason 注明是兼容/限额行陈旧，落地 5a L673「兼容陈旧经所属 plan 复核」的意图，覆盖 tech-plan「兼容矩阵最易过时」的死角）。**`last_checked IS NULL`（从未核对，如 manual/needs_login 占位）= 最该复核**，判定为 `last_checked IS NULL OR last_checked < threshold`，不被静默跳过。`ponytail:` 五表各 seq scan（低百行，B-tree 索引过早）；行数破万再加 `idx_<t>_last_checked`（`ASC NULLS FIRST` 使 NULL 集 + `< threshold` 段单次前向扫，避开 OR-NULL 非 sargable 陷阱）。

### 安全（抓取子系统）

**D10：SSRF 防护（单一 chokepoint，page + robots.txt + 任何派生 URL 抓取都必过）。** ① scheme 仅 `http`/`https`（其它一律拒，含 `file://`/`gopher://`）；② host 过**每厂商域名白名单 = checked-in 常量 `MR_SOURCE_DOMAIN_ALLOWLIST`**（仿 `REVIEW_TRIGGER_KEYWORDS`，PR 评审；**独立于 `source_url`**——`mr_vendors` 仅 `id/normalized_name/name` 无域名列，「从 mr_vendors 派生」不可实现，且从录入的不可信 URL 自取 host 是自授权循环）——录入时 Zod 拒 registrable domain ∉ 白名单的 `source_url`（**仅对 `fetch_strategy∈{http,browser}` will-fetch 源**；`manual` 源不发请求、豁免该闸，URL 仅人类参考），**抓取时再验**；③ 解析 host，命中私网/环回/link-local（`127/8`、`10/8`、`172.16/12`、`192.168/16`、`169.254/16`、`::1`、`fc00::/7`、`fe80::/10`）一律拒；④ **DNS-rebind 闭合按档分层（load-bearing）**：**`http` 档**（Node 驱动 fetch）用 `node:https`/`node:http` 的**原生 `lookup` 选项**——自定义 lookup 解析 host→**全部 A/AAAA**、**任一命中私网即整集拒**（防 rebind 竞态）、否则返回预验 public IP 使 check==connect（**lookup 抛错/解析空集/CNAME-only 无 A·AAAA → `callback(err)` fail-closed 拒连，绝不以默认/陈旧地址放行**）；**仅覆盖 `lookup` 不重写 URL→IP**（保留原 hostname 供 SNI/证书校验，免 MITM 回归）；每重定向跳重解析重验。**node 原生、无需新依赖**（不引 undici、不动全局 `fetch`/`github.ts`）；纯 `dns.lookup`-then-`fetch` 因二次解析挡不住 TOCTOU **明确排除**。**`browser` 档渲染器 socket 不经 Node dispatcher，私网封锁靠网络层 egress，见 D11。**⑤ 重定向 **`redirect:'manual'` + 每跳重跑全套守卫（scheme+白名单+IP）+ 最大跳数上限**（禁 `redirect:'follow'`）。**`robots.txt` 取用必走同一 chokepoint + 响应体上限/超时**（第一个触达不可信 host 的请求、最易漏挡）。

**D11：Playwright 沙箱锁定 + browser 档私网封锁靠网络层 egress（权威）。** browser 档运行**非 root**、Chromium **sandbox 启用**（禁传 `--no-sandbox`）；**每 job 全新 `browser.newContext()`** 用后即关（非复用单 context，防跨源污染）；禁下载（`acceptDownloads:false`）/file chooser/对话框（自动 dismiss）/新窗口/service worker/默认权限。**关键：Chromium 渲染器的导航/子资源/WebSocket socket 不经 Node 的 `lookup`（D10④）——故 browser 档对私网/元数据 IP 的权威封锁必须是网络层 egress 控制**：browser-worker 独立 service（D15）跑在**封 RFC1918/link-local/`169.254.169.254` 的 egress 代理或容器 netns** 内（**必需部署控制，非可选**）。**+ 启动 fail-closed 自检**（仿全局 `doctor` 哲学，把 egress 从运维指令升为运行时门）：`browser-worker-main.ts` 启动时驱浏览器对**每段一个代表哨兵**（元数据 `169.254.169.254` + 一个 RFC1918 如 `10.0.0.1` + 环回 `127.0.0.1`）各发一次请求，**任一未被挡即拒绝启动**（非零退出、不消费 job）——防偏配 egress（只挡元数据却漏 `10/8`/`127/8`）过门、防未配 egress 裸奔成 SSRF-to-metadata 洞。**netns 部署（egress 失效即 fail-closed）是保门选择**；egress 代理可中途挂（自检只证 T0），须配 liveness 重探。`context.route`（URL-string 过滤）+ CDP `Network.setBlockedURLs(['ws://*','wss://*'])` 封 WebSocket = **纵深防御**（页内 JS shim 改 `window.WebSocket` 可被 iframe/Worker/抢跑绕过，**不作权威**）。**硬超时杀进程** = 外层 `Promise.race([job, hardTimeout])` watchdog，超时 `process.kill(browser.process().pid, 'SIGKILL')`（进程树 kill，非 `browser.close()`——后者会被挂死渲染器拖住）+ 容器 cgroup 内存上限 + 最大响应体/重定向数上限防 OOM。

**D12：裸请求（无凭据，机制非纪律）。** Model Radar 抓取 fetch wrapper **不接受任何 `headers`/`token` 参数**（仅固定 `{ 'User-Agent': MR_SCRAPE_UA }`）——无参数可穿凭据，结构上杜绝 `Authorization`/`Cookie`/provider API key；**不得**从带凭据的 collector（如 `github.ts` 带 `GITHUB_TOKEN`）派生 fetch。测试断言出站头 `===` 恰为 `{User-Agent}`。强化既有「不调各家 API」= 也不带任何 API 凭据。**结构守卫**：`src/mr/scrape/` eslint 禁裸调出站原语绕过 SSRF wrapper——`no-restricted-globals`/`no-restricted-properties` 封 `fetch`/`globalThis.fetch`，`no-restricted-imports` 封 `node:http(s)`/`node:net`/`node:dgram`（含 `require` 形式），仿 D7 把「必过 chokepoint」从纪律升为 lint。

**D13：快照安全 + 5c 边界。** 抓取文本快照（供人 diff，**best-effort 证据非权威**——flag 不依赖它存活、过期则人重抓）：存储键 = **安全派生 id**（`sha256(source_id)`），**禁止**把 `source_url`/厂商名拼进路径（防穿越）。**介质 5b 定为短期临时文件**：**原子写**（写 `id.tmp`（与目标同 base-dir 保 rename 同文件系统原子）+ `rename`、处理 ENOSPC，并发同源 last-writer-wins 可接受、绝不让 diff 读到半截文件）+ base-dir 越界断言 `path.resolve(base,id).startsWith(base+sep)` + 短 TTL + **janitor**（陈旧度/cron 扫删过期快照 + 总字节上限防本地 DoS）。以**不可执行字节**存（`text/plain`/`octet-stream`，非内联 `text/html`）；**`nosniff`/`attachment` 标记与「渲染当不可信文本转义」是 5c/5d 渲染边界控制**（落盘静态文件上 nosniff 是 no-op）——5b 自身控制 = **静态不可执行字节 + 5b 无渲染路径**。**`mr_price_history` 只记人工确认后的价格事实历史，不作 5c raw snapshot 表**；5c 如需 raw snapshot 应新建独立 `mr_price_snapshots`/`mr_source_snapshots`（`source_id/snapshot_type/raw_payload/captured_at`）+ 可选迁对象存储。5b 快照文本不入任何 `mr_*` 列、不引新依赖。

### 运营

**D14：BullMQ 四件套完整命名 + 重试/死信。** 每个 5b 队列给齐 `*_QUEUE`/`*_JOB` 常量 + `create*Worker` 函数 + `schedule*` + payload shape（对齐 `alert-queue.ts`）：`MR_EVENT_REVIEW_QUEUE/JOB`、`MR_SCRAPE_HTTP_QUEUE/JOB`、`MR_SCRAPE_BROWSER_QUEUE/JOB`、`MR_STALENESS_QUEUE/JOB`。每队列 `defaultJobOptions{attempts, exponential backoff, removeOnComplete, removeOnFail}`；重试耗尽**保留 failed job**（或投 `mr-*-dead`）供人工排查/重放，**失败不改事实**。

**D15：Playwright worker 独立部署。** browser 抓取是**独立 entrypoint（`browser-worker-main.ts`）+ 独立 compose service + 独立镜像（装 playwright + 浏览器）**——**不**在 `worker-main.ts` 内 `new Worker`（否则 Playwright 进主镜像，违背「主镜像不装」）。与 [[deployment-containerized]] 的单 GHCR 镜像约定的关系在部署时决断（可能扩出第二镜像）。

**D16：`mr_catalog_version` 5b 不 bump（留 5c）。** 5b 的 `upsert*` 仅写 `mr_*`、**不 bump version、不投 rebuild**（catalog spec L105「5a 仅建表不被写路径触碰」+ tech-plan 划归 5c）；但 `upsert*`/`recordPriceChange` 设计为**接受已开外层 tx 句柄**（D4 内部 helper 收 `TxLike`，非 `DbLike|TxLike`——多语句 + `FOR UPDATE` 须真事务）+ 返回受影响结果，使 5c 能把 fact-write + `mr_catalog_version` bump + enqueue 裹进**同一外层事务**（避免 commit 后再 bump 的版本漂移窗口），无须重构 5b 写函数。

## 风险 / 权衡

- SSRF 白名单/私网封锁 = 抓取安全底线（非新增 feature，是 in-scope 抓取行为的安全硬化）；DNS rebind 的 TOCTOU **按档闭合**——`http` 档用 `node:https` 原生 lookup（连预验 IP、整集任一私网即拒），`browser` 档渲染器 socket 靠网络层 egress（D11，必需部署控制）。错误/日志只记**通用枚举原因 + 已验证 source id**，绝不记解析到的 IP/原始连接错误/重定向目标（防经差异化报错把内网拓扑当 oracle 探测），明细只进访问受控的快照。
- Playwright 沙箱/隔离运维成本 → 独立镜像 + 每 job 新 context；挂死渲染器用杀进程兜底。
- 事件流误召/漏召 → 误召只多打标无害；漏召由 D9 全表陈旧度兜底（含 junction/NULL）。
- 单写者策展不上乐观版本列（D2）→ 多写者出现再加，避免过早复杂度。

## 迁移计划

默认**不新增迁移**（复用 5a schema + 词表扩值不改语义）。Playwright 作独立镜像/worker；CI/部署 browser worker 单独 service。验证：录入/CAS/append-only/事件/陈旧度用集成测试（真实 pg + 注入桩，不触网）；抓取链用注入 fetch/Playwright 桩 + SSRF/沙箱单测（不真抓外网）；真实抓取勘验交付用户跑一次。
