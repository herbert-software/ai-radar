## 1. Drizzle schema 定义（`src/db/schema.ts` 追加 `mr_*`）

> 通用约定：**PK = `id varchar(128) PRIMARY KEY DEFAULT gen_random_uuid()::text`**（对齐 `ai_products`/`ai_news_events`/`ai_experiences` 代理键惯例，使 `mr_review_flag.target_id` 可统一引各身份表 PK）；引用列裸 id（`varchar(128)`，无 FK）；**每张唯一键组件列一律 `.notNull()`**（PG `UNIQUE` 对 NULL distinct，组件可空会让去重键失效）；每表带 `created_at timestamptz NOT NULL DEFAULT now()`。

- [x] 1.1 `mr_vendors`（`id` PK、`normalized_name text NOT NULL`、`name text NOT NULL`、`created_at`；`UNIQUE(normalized_name)`）+ `mr_models`（`id` PK、`vendor_id varchar(128) NOT NULL`、`family text NOT NULL`（小写）、`version text NOT NULL`（哨兵 `''`）、`created_at`；`UNIQUE(vendor_id, family, version)`；身份行不挂 provenance）
- [x] 1.2 `mr_plans`（`id` PK、`vendor_id varchar(128) NOT NULL`、`name text NOT NULL`、`category text NOT NULL`、`current_price numeric(12,2)` **NULL**、`currency varchar(3)` **NULL**（与 current_price 同生同灭，由 1.7 Zod refine 兜）、provenance `source_url text NOT NULL`/`last_checked timestamptz NOT NULL`/`source_confidence text NOT NULL`、`created_at`、`updated_at timestamptz NOT NULL DEFAULT now()`；`UNIQUE(vendor_id, name)`）
- [x] 1.3 junction（两张都挂 provenance 三字段 NOT NULL）：`mr_plan_models`（`id` PK、`plan_id varchar(128) NOT NULL`、`model_id varchar(128) NOT NULL`、`source_url`/`last_checked`/`source_confidence` NOT NULL、`created_at`；`UNIQUE(plan_id, model_id)`）与 `mr_plan_clients`（`id` PK、`plan_id NOT NULL`、`client_type text NOT NULL {tool,protocol}`、`client_id text NOT NULL`、provenance NOT NULL、`created_at`；`UNIQUE(plan_id, client_type, client_id)`）；反向索引暂缓，注释记理由
- [x] 1.4 `mr_plan_limits`（`id` PK、`plan_id NOT NULL`、`limit_type text NOT NULL`、`value numeric` **NULL**、`window text NOT NULL`（哨兵 `'5h'`/`'week'`/`'month'`/`'none'`）、provenance NOT NULL、`created_at`、`updated_at timestamptz NOT NULL DEFAULT now()`；`UNIQUE(plan_id, limit_type, window)`）。`value` numeric 禁 integer
- [x] 1.5 `mr_price_history`（`id` PK、`plan_id NOT NULL`、`old_value numeric(12,2)` NULL、`new_value numeric(12,2) NOT NULL`、`currency varchar(3) NOT NULL`、`changed_at timestamptz NOT NULL`、provenance 例外（`source_url NOT NULL`/`source_confidence NOT NULL`，`changed_at` 兼任 last_checked，**不建 last_checked**）、`created_at`；`UNIQUE(plan_id, changed_at)`，append-only）
- [x] 1.6 接缝表：`mr_source`（`id` PK、`source_url text NOT NULL`、`vendor_id NOT NULL`、`fetch_strategy text NOT NULL`、`content_fingerprint text` NULL、`last_checked timestamptz` **NULL 无 default**（未抓过 ≠ 入库时刻）、`created_at`；`UNIQUE(vendor_id, source_url)`）+ `mr_plan_sources`（`id` PK、`source_id NOT NULL`、`plan_id NOT NULL`、`created_at`；`UNIQUE(source_id, plan_id)`）+ `mr_review_flag`（`id` PK、`target_type text NOT NULL {plan,source,vendor}`、`target_id varchar(128) NOT NULL`、`reason text` NULL、`status text NOT NULL {pending,resolved}`、`opened_at timestamptz NOT NULL DEFAULT now()`、`resolved_at timestamptz` NULL、`created_at`；`UNIQUE(target_type, target_id)`，普通约束非 partial index）+ `mr_catalog_version`（`id` PK、`version bigint NOT NULL`（无 default，由 5c bump 路径产生，5a 不插）、`built_at timestamptz NOT NULL DEFAULT now()`、`created_at`；`UNIQUE(version)`；5a 仅建表不写）
- [x] 1.7 Zod（应用层，DB 不建 CHECK/pg-enum）：枚举 `source_confidence`（5 值）、`category`（恰好 4 值）、`fetch_strategy`（3 值）、`currency`（`z.enum(['CNY','USD'])` 大写，5b 扩集）、`limit_type`（桶2 样例集 `monthly_tokens`/`rolling_5h_requests`/`weekly_messages`/`none`，credit/fast_pass 留 5b；不声称覆盖 8 家全部）、`mr_review_flag.status`（`{pending,resolved}`）、`mr_review_flag.target_type`（`{plan,source,vendor}`）、`mr_plan_clients.client_type`（`{tool,protocol}`）——**所有有限值集列都必须有 Zod 枚举闸**（零-CHECK 下这是唯一合法性防线）；**`mr_plans` schema 加 `.refine()` 断言 `(current_price==null)===(currency==null)`**（同生同灭）；`family` 小写归一 = 5b 录入契约（注释，无 transform）
